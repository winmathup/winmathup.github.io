39.2.3 class를 이용한 요소 노드 취득

Element.prototype.getElementsByClassName 메서드

class값은 공백으로 구분하여 여러 개의 class를 지정할 수 있다.

[...$elems].forEach(elem => {elem.style.color='red';});

  element.style.backgroundColor = 'yellow';

  element.style.cssText  = 'color: blue; background-color: yellow';

[...$elems].forEach(elem => { elem.style.cssText = 'color:blue;background-color:yellow' });

style 속성과 style.cssText 속성을 사용하여 style을 지정할 때 주의해야 할 것이 있습니다.

  • style.css속성명 - 기존에 정의된 style에 새로운 속성을 추가합니다.
  • style.cssText - 기존에 정의된 style을 지우고, cssText로 덮어 씁니다.

style.fontWeight=’bold’를 지정하였지만 element.style.cssText 구문이 앞에서 지정한 ‘fontWeight’ 속성을 모두 덮어쓴 것을 확인 할 수 있습니다.

 element.style.cssText  = 'color: blue; background-color: yellow';

  // 3. Style 속성 지정(추가된다)
  element.style.fontWeight='bold';
}

Window.getComputedStyle() 메소드는 
파라미터로 전달받은 요소의 모든 CSS 속성값을 담은 객체를 리턴합니다.

  const bgColor = getComputedStyle(element).backgroundColor;

39.2.4 CSS선택자를 이용한 요소 노드 취득

CSS선택자는 스타일을 적용하고자 하는 HTML 요소를 특정할 때 사용하는 문법이다.

  • p { … } : p 태그 요소 모두 선택

  • #foo { … } : id ‘foo’ 요소 모두 선택

  • .foo { … } : class ‘foo’ 요소 모두 선택

  • input[type=text] { … } : input 요소 중에 type 어트리뷰트 값이 text인 요소 모두 선택

  • div p { … } : div 요소의 후손 요소 중 p 요소를 모두 선택

  • div > p { … } : div 요소의 자식 요소 중 p 요소를 모두 선택

  • p + ul { … } : p 요소의 형제 요소 중에 p 요소 바로 뒤에 위치하는 ul 요소를 선택

  • p ~ ul { … } : p 요소의 형제 요소 중에 p 요소 뒤에 위치하는 ul 요소를 모두 선택

  • a:hover { … } : hover 상태인 a 요소를 모두 선택

  • p::before { … } : p 요소의 콘텐츠의 앞에 위치하는 공간을 선택 일반적으로 content 프로퍼티와 함께 사용된다.

querySelector, querySelectorAll 메서드는 getElementById, getElementsBy*** 메서드보다 다소 느리다. 하지만 CSS 선택자 문법을 사용하여 좀 더 구체적인 조건으로 요소 노드를 취득할 수 있고 일관된 방식으로 요소 노드를 취득할 수 있다는 장점이 있다.따라서 id 어트리뷰트가 있는 요소 노드를 취득하는 경우에는 getElementById 메서드를 사용하고 그 외의 경우에는 querySelector, querySelectorAll 메서드를 사용하는 것을 권장한다.

39.2.6 HTMLCollection 과 NodeList

1 . HTMLCollection

HtmlCollection은 항상 live 객체로 동작한다. NodeList는 대부분의 경우 노드 객체의 상태변화를 실시간으로 반영하지 않고 과거의 정적 상태를 유지하는 non-live 객체로 동작하지만 경우에 따라 live 객체로 동작할 때가 있다.

<!DOCTYPE html>
<hdad>
    <style>
        .red {
            color: red;
        }

        .blue {
            color: blue;
        }
    </style>
</hdad>

<html>

<body>
    <ul id="fruits">
        <li class="red">Apple</li>
        <li class="red">Banana</li>
        <li class="red">Orange</li>
    </ul>
    <script>
        const $elems = document.getElementsByClassName('red');
        console.log($elems);
        for (let i = 0; i < $elems.length; i++) {
            $elems[i].className = 'blue';
        }
        console.log($elems);
    </script>

</body>

</html>

i=0일 때 : elems[0] red에서 blue로 바뀌고 제거됨 i=1일 때 : length3에서 length2로 바뀜, elems[1]은 제거되기전 elems[2]와 같음, 위와 같이 바뀌고 제거됨 i=2일 때 : length2에서 length1로 바뀌며 elems는 elems[0]만(초기 elems[1]값) elems[2]를 가리키므로 false로 종료됨. 오류 방지하기 위해 for문을 역방향으로 하는 방법이 있다.

2 . NodeList

HtmlCollection 객체의 부작용을 해결하기 위해 getElementsByTagName, getElementsByClassName 메서드 대신 querySelectorAll 메서드를 사용하는 방법도 있다. 단 실시간으로 객체의 상태 변경을 하지 않는다.하지만 childNodes 프로퍼티가 반환하는 NodeList 객체는 HTMLCollection 객체와 같이 실시간으로 노드 객체의 상태변경을 반영하는 live 객체로 동작한다.

39.3.1 공백 텍스트 노드

HTML 요소 사이의 스페이스, 탭, 줄바꿈(개행) 등의 공백 문자는 텍스트 노드를 생성한다.

39.3.2 자식 노드 탐색

노드 탐색 프로퍼티 사용

  1. Node.prototype.childNodes : 자식노드를 모두 탐색하여 DOM 객체인 NodeList에 담아 반환한다. childNodes 프로퍼티가 반환한 NodeList에는 요소 노드뿐만 아니라 텍스트 노드도 포함되어 있을 수 있다.

  2. Element.prototype.children : HTMLCollection에 담아 반환한다. children 프로퍼티가 반환한 HTMLCollection에는 텍스트 노드가 포함되지 않는다.

  3. Node.prototype.firstChild : 첫번째 자식 노드를 반환한다.

  4. Node.prototype.lastChilde : 마지막 자식 노드를 반환한다.

  5. Element.prototype.firstElementChild : 첫번째 자식 요소 노드를 반환한다.

  6. Element.prototype.lastElementChild : 마지막 자식 요소 노드를 반환한다.

39.3.3 자식 노드 존재 확인

Node.prototype.hasChildNodes 메서드 사용한다. 자식 노드가 존재하면 true, 존재하지 않으면 false

자식 노드 중에 텍스트 노드가 아닌 요소 노드가 존재하는지 확인하려면 hasChildNodes 메서드 대신 children.length 또는 Element 인터페이스의 childElementCount 프로퍼티를 사용한다.

39.3.5 부모 노트 탐색

부모 노트 탐색하려면 Node.prototype.parentNode 사용. 텍스트 노드는 DOM 트리의 최종단 노드인 리프 노드이므로 부모노드가 텍스트 노드인 경우는 없다.

39.4 노드 정보 취득

  1. Node.prototype.nodeType

    • Node.ELEMENT_NODE(1반환)

    • Node.TEXT_NODE(3반환)

    • Node.DOCUMENT_NODE(9반환)

  2. Node.prototype.nodeName

    • 요소 노드 : UL, LI 반환

    • 텍스트 노드 : #text 반환

    • 문서 노드 : #document 반환

39.5.1 nodeValue

Node.prototype.nodeValue 프로퍼티는 setter와 getter 모두 존재하는 접근자 프로퍼티다. 참조와 할당 모두 가능하다. 노드 객체의 값을 반환하는데 노드 객체의 값이란 텍스트 노드의 텍스트다. 따라서 텍스트 노드가 아닌 노드인 문서노드나 요소노드의 nodeValue 프로퍼티를 참조하면 null을 반환한다.
